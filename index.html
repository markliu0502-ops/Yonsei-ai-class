제작 프로세스

Creation Process

Step 1

기획 및 구조 설계

Planning & Structure

코드를 작성하기 전, 페이지의 목적과 타겟을 명확히 하는 것이 중요합니다. AI에게 구체적인 컨셉을 제시하여 페이지에 들어갈 텍스트와 레이아웃 구조를 먼저 추천받으세요.

Before coding, it's crucial to define the purpose and target audience. Give the AI a specific concept to get text recommendations and layout structures.

Representative Prompt "새로운 SaaS 서비스를 위한 랜딩 페이지를 만들려고 해. 헤더, 주요 기능 3가지, 요금제, 푸터로 구성된 깔끔한 원페이지 구조와 카피라이팅을 제안해줘." "I'm building a landing page for a new SaaS. Suggest a clean one-page structure including a header, 3 key features, pricing, and footer, along with copywriting."
Step 2

HTML/CSS 디자인 및 구현

HTML/CSS Design & Implementation

기획된 내용을 바탕으로 실제 코드를 작성합니다. 초보자라면 여러 파일을 나누기보다 하나의 HTML 파일에 CSS를 포함하는 방식이 테스트하기 쉽습니다. 모바일 반응형 디자인 조건도 잊지 마세요.

Write the actual code based on the plan. For beginners, it's easier to include CSS within a single HTML file for testing. Don't forget to ask for mobile responsive design.

Representative Prompt "앞서 제안해준 내용을 바탕으로 미니멀한 디자인의 원페이지 HTML/CSS 코드를 작성해줘. 검정색과 흰색을 메인 테마로 사용하고, 스마트폰에서도 잘 보이도록 반응형으로 만들어줘." "Write a one-page HTML/CSS code with a minimal design based on your suggestion. Use black and white as the main theme, and make it fully responsive for mobile devices."
Step 3

GitHub Pages로 배포하기

Deploying with GitHub Pages

완성된 코드는 무료 호스팅 서비스인 GitHub Pages를 통해 전 세계에 공개할 수 있습니다. 로컬 이미지나 비디오를 사용할 때는 반드시 상대 경로(`./assets/파일명`)를 사용해야 링크가 깨지지 않습니다.

You can publish your completed code globally using GitHub Pages, a free hosting service. When using local images or videos, make sure to use relative paths (`./assets/filename`) to avoid broken links.

Representative Prompt "작성한 index.html 파일과 assets 폴더를 GitHub Repository에 올리고, GitHub Pages를 활성화해서 무료 웹사이트 링크를 얻는 과정을 스텝별로 알려줘." "Explain step-by-step how to upload my index.html and assets folder to a GitHub Repository and enable GitHub Pages to get a live website link."
